Three directions for modernizing the algo editor. Open each variant, compare the chrome, hierarchy, and feel — then tell me which direction (or which parts of which) to build.

Variant A

Minimal · Linear-inspired

Restrained chrome. Contextual breadcrumb header replaces the static "Algo Editor" label. Footer status bar carries saved-state, cursor position, and deps count.

  • Unsaved dot in sidebar + header
  • Autosave with visible state
  • Overflow ⋯ menu, not button row
  • Subtle sidebar search

Variant B

Workspace · VS Code-inspired

Editor tabs. Activity bar on the far left. Multiple algos open at once. Richer status bar. Familiar to anyone who's used VS Code or Cursor.

  • Tabs with dirty dot + close
  • Activity bar for future panels
  • Breadcrumb under tabs
  • ⌘K command palette affordance

Variant C

Focus · Writer-inspired

Max editor real estate. Sidebar collapses to a thin rail; peeks open on hover. Floating ⌘K palette is the primary nav. For people who know their algos by name.

  • Collapsed rail with just algo count
  • Hover-peek sidebar
  • Floating command palette
  • Minimalist top bar

Notes
These are visual prototypes only. The editor surface is a hand-rolled lookalike — the real implementation would swap in Monaco with a custom theme that reads your --bg-panel, --border, and accent tokens (none of the variants use vs-dark).

Every button, tab, and search box is static — clicking around the app isn't wired up. Focus on the layout, hierarchy, density, and chrome, not the interactions. Flip between variants with the banner in the top right of each page.
